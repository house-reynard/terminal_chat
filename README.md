A simple terminal-first chat application built with Go, PostgreSQL, WebSockets, and Bubble Tea.

Terminal Chat is a personal learning project created to explore real-time messaging systems, WebSocket communication, database design, and terminal user interface development. Instead of building yet another web application, I wanted to create something that lives entirely in the terminal and embraces a keyboard-driven workflow.

Inspired by tools like Yazi, a fast terminal file manager designed for keyboard-centric users, Terminal Chat aims to provide a lightweight messaging experience without relying on browsers or traditional desktop applications.

## Tech Stack

* **Go** - Backend and client implementation
* **PostgreSQL** - Data persistence
* **WebSockets** - Real-time communication
* **Bubble Tea** - Terminal user interface framework
* **Lip Gloss** - Terminal styling

## Why This Project?

As someone who spends most of their time inside the terminal, I wanted to understand how modern chat applications work under the hood while building something I would actually enjoy using.

This project is primarily focused on learning:

* WebSocket communication
* Real-time messaging architecture
* Client-server design
* PostgreSQL database integration
* Authentication and user management
* Terminal User Interfaces (TUI)
* Concurrent programming in Go
* Network programming concepts

## Features

### Current Goals

* Terminal-based chat client
* Real-time messaging using WebSockets
* PostgreSQL-backed message storage
* Keyboard-only navigation
* Multiple chat rooms/channels
* User authentication
* Cross-platform terminal support

### Planned

* Linux-first installation experience
* Windows support
* Improved TUI experience
* Better message history handling
* Configuration management
* Quality-of-life terminal features

## Philosophy

Terminal Chat is intentionally simple. It serves as a practical environment for experimenting with networking concepts, messaging systems, and terminal application development.

## Status

This project is currently under active development.

The idea for Terminal Chat comes from a love of keyboard-driven workflows.

Tools like Yazi demonstrate how powerful terminal experiences can be when designed thoughtfully. This project applies that same terminal-first mindset to real-time communication and messaging.
